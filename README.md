## 📖 About the Project

> **3D Café** is a premium full-stack food ordering platform built with **React 18, TypeScript, Node.js, and MongoDB** — delivering an immersive dining experience through interactive 3D product visualisation, real-time order tracking, and a complete enterprise ERP system for café management.

Most food ordering platforms are simple menus with a cart. **3D Café** goes far beyond — it's a production-grade ecosystem with a customer-facing storefront, real-time kitchen queue, admin ERP, CRM, full HR management, Excel import/export, and automated CI/CD pipelines.

This project was built as a **portfolio-grade full-stack application** demonstrating senior-level architecture decisions, scalable backend design, TypeScript discipline, and modern DevOps practices — all deployed and live on Render.

## 🏗️ Architecture

```
Browser / Client
       │
       ▼
React SPA (Vite + TypeScript)
  ├── Redux Toolkit  →  Auth, Cart, UI state
  ├── React Query    →  Server cache (food, orders, analytics)
  ├── Socket.io      →  Live order feed, kitchen queue
  └── Three.js       →  3D product viewer
       │
       ▼  REST API + WebSocket
Express.js Server
  ├── JWT Middleware  →  Access + Refresh token rotation
  ├── Role Guards     →  customer / staff / admin
  ├── Rate Limiting   →  per-route brute-force protection
  ├── Input Validation → express-validator + mongo-sanitize
  └── Socket.io       →  Event broadcasting
       │
       ▼
MongoDB (Mongoose)
  ├── Users  (JWT, OAuth, loyalty, addresses)
  ├── FoodItems (3D assets, categories, pricing)
  ├── Orders (items, status history, payments)
  ├── Inventory (stock, suppliers, purchase orders)
  ├── Coupons (discount rules, expiry)
  └── Reviews (per-item ratings)
       │
       ▼
Optional Integrations
  ├── Cloudinary   →  Image upload + CDN
  ├── Stripe       →  Payment processing
  ├── Nodemailer   →  Email verification + reset
  └── Google OAuth →  Social sign-in
```

## ⚙️ Installation

**Step 1 — Clone the repository**

```bash
git clone https://github.com/vinushinde2525-sys/3D-Cafe-FullStack-Platform.git
cd 3D-Cafe-FullStack-Platform
```

**Step 2 — Install dependencies**

```bash
# Frontend
cd frontend
npm install

# Backend
cd ../backend
npm install
```

**Step 3 — Configure environment**

```bash
# Backend
cp backend/.env.example backend/.env
# Required: MONGODB_URI, JWT_SECRET, JWT_REFRESH_SECRET
# All other integrations (Google OAuth, Stripe, Cloudinary, Email) are optional

# Frontend
cp frontend/.env.example frontend/.env
# Set VITE_DEMO_MODE=true for no-backend demo mode
```

**Step 4 — Run**

```bash
# Backend (from /backend)
npm run dev       # http://localhost:5000

# Frontend (from /frontend)
npm run dev       # http://localhost:5173
```

## ⚡ Performance & Architecture Decisions

| Decision | Reason |
|---|---|
| Redux for auth + cart | Client state that persists across pages and needs cross-component access |
| React Query for API data | Automatic caching, background refetch, deduplication — no manual loading state |
| JWT rotation on every request | Prevents token theft — compromise window is one request cycle |
| Socket.io room-based events | Orders broadcast only to relevant kitchen/customer rooms — no polling |
| Vitest `vmThreads` pool | Solves Windows worker spawn timeout with heavy imports (Three.js, Redux) |
| Demo mode via localStorage | App is fully usable for recruiters without a running backend |
| Vite manual chunks | Splits Three.js (~800KB), Framer Motion, Redux into separate cache-busted bundles |

---

## 🧩 Challenges & Solutions

| Challenge | Solution |
|-----------|----------|
| 3D viewer causing layout reflows | Wrapped in `Suspense` + skeleton; Three.js chunk separated via Vite `manualChunks` |
| JWT expiry causing silent logouts | Auto-refresh token rotation in `protect` middleware — seamless re-auth without user action |
| Real-time updates causing re-render loops | Socket hooks memoized with `useCallback`; listeners attached once via `useEffect` cleanup |
| Windows Vitest worker timeout | Switched to `vmThreads` pool — resolves fork spawn hang on Windows with heavy deps |
| Backend optional integrations | All 4 integrations (Google/Stripe/Cloudinary/Email) degrade gracefully with clear console logs |
| Cart state persistence | localStorage sync in Redux reducer — cart survives page refresh without backend |
