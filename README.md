## 📖 About the Project

> **3D Café** is a production-grade, full-stack food ordering platform built with the **MERN stack** — crafted to deliver an immersive café experience through interactive 3D visuals, real-time order tracking, and role-based dashboards for customers, admins, and staff.

This project was built as a **portfolio-grade application** demonstrating end-to-end full-stack development: modern React + TypeScript architecture, real-time Socket.io communication, secure JWT authentication with RBAC, Stripe-ready payment flow, and Docker-containerised deployment.

The frontend runs fully in **Demo Mode** — no backend required — making it instantly deployable to Vercel or Netlify.

## 🔌 API Endpoints

<details open>
<summary>🍽️ Menu</summary>

```http
GET    /api/menu              # All items (filter & search)
GET    /api/menu/featured     # Featured items
GET    /api/menu/:id          # Single item detail
POST   /api/menu              # Create item (Admin)
PUT    /api/menu/:id          # Update item (Admin)
DELETE /api/menu/:id          # Delete item (Admin)
```

</details>

<details>
<summary>📦 Orders</summary>

```http
POST   /api/orders            # Place new order
GET    /api/orders            # Customer order history
GET    /api/orders/all        # All orders (Admin/Staff)
PUT    /api/orders/:id/status # Update status (Staff)
```

</details>

<details>
<summary>🔐 Auth</summary>

```http
POST   /api/auth/register         # Register account
POST   /api/auth/login            # Login
POST   /api/auth/refresh          # Refresh token
GET    /api/auth/google           # Google OAuth
GET    /api/auth/google/callback  # OAuth callback
```

## ⚙️ Installation

### ⚡ Option 1 — Frontend Only (Demo Mode)

```bash
git clone https://github.com/YOUR_USERNAME/cafe-3d.git
cd cafe-3d/frontend
npm install
npm run dev
```

✅ Open `http://localhost:5173`

---

### 🗄️ Option 2 — Full Stack

**Step 1 — Backend env**

```bash
cp backend/.env.example backend/.env
```

```env
MONGODB_URI=your_mongodb_uri
JWT_SECRET=your_jwt_secret
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
STRIPE_SECRET_KEY=your_stripe_secret
GOOGLE_CLIENT_ID=your_client_id
GOOGLE_CLIENT_SECRET=your_client_secret
```

**Step 2 — Start backend**

```bash
cd backend && npm install
npm run seed
npm run dev
```

✅ Backend: `http://localhost:5000`

**Step 3 — Start frontend**

```bash
cd frontend
cp .env.example .env.local
# set VITE_API_URL=http://localhost:5000/api
npm install && npm run dev
```

✅ Frontend: `http://localhost:5173`

---

### 🐳 Option 3 — Docker

```bash
# Development
docker-compose --profile dev up --build

# Production
docker-compose up --build -d
```
